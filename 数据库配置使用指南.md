**系统如何访问这些表：**

在 `backend/models/models.py` 中，数据库连接时会自动设置 search_path：

```python
def create_engine_and_session():
    db_type = DB_CONFIG.get('db_type', 'postgresql')
    
    if db_type == 'postgresql':
        # PostgreSQL 连接 URL
        db_url = f"postgresql://{DB_CONFIG['user']}:{DB_CONFIG['password']}@{DB_CONFIG['host']}:{DB_CONFIG['port']}/{DB_CONFIG['database']}"
    
    engine = create_engine(
        db_url, 
        connect_args={
            'options': f'-csearch_path={DB_CONFIG.get("schema", "public")}'
        } if db_type == 'postgresql' else {}
    )
```

**解释：**
- `search_path` 参数告诉 PostgreSQL 在哪个 Schema 中查找表
- 设置为 `crawled-zxb-data` 后，SQL 查询会自动在该 Schema 下查找 `questions`、`replies`、`images` 表
- 无需在代码中显式指定 Schema

## 🗂️ 数据库表结构映射

### 您数据库中的表（PostgreSQL Schema: crawled-zxb-data）

根据截图，您的数据库表结构如下：

```sql
-- Schema: crawled-zxb-data

-- 1. 问题表
questions (
    id              -- 主键
    question_no     -- ⭐ 问题编号（唯一标识）
    title           -- 问题标题
    content         -- 问题内容
    created_at      -- 创建时间
)

-- 2. 回复表
replies (
    id              -- 主键
    question_no     -- ⭐ 关联问题编号
    content         -- 回复内容
    accepted_flag   -- ⭐ 采纳状态（0/1）
    user_type       -- ⭐ 用户类型（1/2/3）
    evl             -- ⭐ 评分（系统自动计算）
    created_at      -- 创建时间
)

-- 3. 图片表
images (
    id              -- 主键
    entity_id       -- ⭐ 关联实体 ID（对应 question_no）
    url             -- ⭐ 图片 URL
    created_at      -- 创建时间
)
```

### 表之间的关联关系

```
questions (question_no)
    ↓
    ├─→ replies (question_no)  ← 一对多关系
    └─→ images (entity_id)     ← 一对多关系
```

**关键点：**
- `questions.question_no` = `replies.question_no`
- `questions.question_no` = `images.entity_id`
- 一个问题是多个回复
- 一个问题可以有多张图片

## 💾 数据存储流程

### 1. Excel 文件上传时的数据存储

**流程步骤：**

1. **Excel 文件解析**
   - 位置：`backend/utils/excel_parser.py`
   - 功能：解析用户上传的 Excel 文件，提取问题、回复和图片数据

2. **数据存储到数据库**
   - 位置：`backend/routes/api.py` 的 `upload_excel` 函数
   - 功能：
     - **问题表存储**：将解析的问题数据存储到 `crawled-zxb-data.questions` 表
     - **回复表存储**：将解析的回复数据存储到 `crawled-zxb-data.replies` 表
     - **图片表存储**：将解析的图片数据存储到 `crawled-zxb-data.images` 表
   - 存储策略：
     - 批量处理：每批处理 1000 条记录，提高效率
     - 重复检查：如果记录已存在则更新，不存在则插入
     - 错误处理：单条记录失败不影响其他记录处理

### 2. 评分过程中的数据存储

**流程步骤：**

1. **评分计算**
   - 位置：`backend/services/scoring.py` 的 `calculate_reply_score` 函数
   - 功能：根据采纳状态、用户身份和内容详实度计算每条回复的评分

2. **评分存储到回复表**
   - 位置：`backend/services/scoring.py` 的 `score_all_replies` 函数
   - 功能：
     - 将评分结果存储到 `crawled-zxb-data.replies` 表的 `score` 列
     - 批量更新，每批处理 1000 条记录
     - 支持事务处理，确保数据一致性

3. **最佳回复选择与存储**
   - 位置：`backend/services/scoring.py` 的 `get_best_reply_for_question` 函数
   - 功能：
     - 为每个问题选择评分最高的回复
     - 将最佳回复和评分存储到 `test.optimize_data` 表
     - 字段映射：
       - `score` → `qa_similarity`（质量分）
       - 包含问题内容、回复内容、优化内容等

### 3. 数据管理页面的数据来源

**数据来源：**
- **数据列表**：从 `test.optimize_data` 表获取数据
- **筛选功能**：支持按审核状态、质量等级和关键词搜索
- **数据更新**：人工审核后，状态会更新到 `test.optimize_data` 表

## 🔍 如何在代码中找到指定的表

### 方法 1：通过 Schema 自动定位（已实现）

系统会自动在 `crawled-zxb-data` Schema 下查找表，无需修改代码。

**验证方法：**
```python
# 在 Python 中测试连接
from backend.models.models import create_engine_and_session

engine, session = create_engine_and_session()

# 查询问题表
from sqlalchemy import text
result = session.execute(text("SELECT * FROM questions LIMIT 5"))
for row in result:
    print(row)
```

### 方法 2：显式指定 Schema（备选方案）

如果需要显式指定 Schema，可以修改模型：

```python
class Question(Base):
    __tablename__ = 'questions'
    __table_args__ = {'schema': 'crawled-zxb-data'}
```

但当前配置已经通过 `search_path` 实现了自动定位，所以不需要这样做。

## 📌 重要提示

1. **PostgreSQL vs MySQL**
   - 您的数据库是 PostgreSQL（端口 5432）
   - 系统已配置为使用 PostgreSQL 驱动（psycopg2-binary）
   - 确保已安装：`pip install psycopg2-binary`

2. **Schema 配置**
   - `search_path` 自动设置，无需在 SQL 中指定 Schema
   - 如果需要在多个 Schema 间切换，修改 `DB_CONFIG['schema']`

3. **依赖安装**
   ```bash
   pip install -r requirements.txt
   ```
   
   这会安装：
   - `psycopg2-binary` - PostgreSQL 驱动
   - `pymysql` - MySQL 驱动（备用）

4. **表名必须匹配**
   - 代码中的表名：`questions`、`replies`、`images`
   - 必须与数据库中的表名完全一致

5. **数据存储注意事项**
   - Excel 上传时，系统会自动创建缺失的表
   - 评分功能会自动更新 `replies` 表的 `score` 列
   - 最佳回复会存储到 `test.optimize_data` 表

## ❓ 常见问题

### Q1: 如何确认当前连接的是哪个 Schema？
**A:** 查看 `backend/config/database.py` 中的 `schema` 配置项，当前是 `crawled-zxb-data`。

### Q2: 如果表不在 crawled-zxb-data 下怎么办？
**A:** 修改 `DB_CONFIG['schema']` 为正确的 Schema 名称。

### Q3: Excel 列名必须和配置一样吗？
**A:** 是的，或者修改 `EXCEL_COLUMN_MAPPING` 配置以匹配您的 Excel 列名。

### Q4: 如何查看配置在哪些地方被使用了？
**A:** 
- 搜索 `DB_CONFIG` 找到数据库连接相关代码
- 搜索 `EXCEL_COLUMN_MAPPING` 找到 Excel 解析相关代码

### Q5: 评分结果存储在哪里？
**A:** 评分结果存储在 `crawled-zxb-data.replies` 表的 `score` 列，同时最佳回复会存储到 `test.optimize_data` 表。

### Q6: 数据管理页面的数据来源是什么？
**A:** 数据管理页面的数据来自 `test.optimize_data` 表，包含问题、回复、质量分和审核状态等信息。
